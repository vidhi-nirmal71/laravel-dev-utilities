Laravel 8.83 to 9.52.15 Upgrade Documentation

This document outlines the detailed steps required to upgrade a Laravel project from version 8.83 to 9.52.15. The upgrade includes updates to the composer.json file, adjustments to key package versions, and modifications to core files like Handler.php for compatibility with Laravel 9.52.15.

Why Laravel 9.52.15 and Not Laravel 9.*?

We have specifically chosen Laravel 9.52.15 and not the latest 9.* series because Laravel 9.52.15 is the final release of the Laravel 9.x line and receives long-term support (LTS). It includes all critical bug fixes, security updates, and performance improvements.

Choosing this version ensures maximum compatibility with our current package ecosystem such as altek/accountant, arcanedev/log-viewer, and others. By locking to this specific version, we avoid potential issues from newer 9.* releases and maintain a secure, stable codebase in production.

1. Backup the Project and Database

# Backup the code
cp -r laravel-adminpanel laravel-adminpanel-backup

# Or use Git
git add .
git commit -m "Backup before Laravel 9 upgrade"
git tag pre-laravel-9
git push origin dev-upgrade-8-to-9 --tags

# Backup the database (MySQL)
mysqldump -u youruser -p yourdatabase > db_backup.sql

2. Update Laravel Version in composer.json

Edit composer.json to:

"laravel/framework": "^9.52.15"

Also verify compatibility of any Laravel-specific packages with Laravel 9.

3. Try to Update Composer Packages


composer update
# If memory errors occur:
COMPOSER_MEMORY_LIMIT=-1 composer update

4. Fix Package Compatibility Issues

  • Problem 1: Package X requires laravel/framework ^8.83
  • Solution: composer require vendor/package:^X.Y
  • Problem 2: Abandoned or incompatible packages
  • Solutions:
    • Check the GitHub repo for Laravel 9 support
    • Replace with an alternative package
    • Temporarily remove it from composer.json
    • Retry composer update

5. Clear and Optimize Laravel Setup


php artisan config:clear
php artisan route:clear
php artisan view:clear
php artisan cache:clear
composer dump-autoload
php artisan optimize

6. Handle Code Changes (e.g., Handler.php)

Laravel 9 introduces changes to core files like app/Exceptions/Handler.php. You should:

  • Adjust method signatures
  • Remove deprecated methods
  • Add missing imports

7. Check Project Functionality

Start the server:

php artisan serve

Ensure:

  • All pages load correctly
  • Forms and submissions work
  • Admin panel, login/logout, dashboard are functional
  • All JS and CSS assets load properly
  • No console or network errors

8. Run Migrations and Test Data


php artisan migrate
php artisan db:seed

9. Run Tests (If Available)


php artisan test

Fix any failing tests.

10. Final Cleanup


php artisan config:cache
php artisan route:cache
php artisan view:cache

# Optional
vendor/bin/php-cs-fixer fix

# Commit (skip GrumPHP if needed)
git commit --no-verify -m "Laravel upgraded to 9.52.15"

Composer Package Changes

1. ✅ Updated Packages:
  • PHP Version:
    Old: ^7.4 → New: ^8.0
    Laravel 9 requires PHP 8.0+, improving performance, error handling, and features.
  • altek/accountant: ^2.1 → ^3.3
    Ensures compatibility with Laravel 9.x
  • arcanedev/log-viewer: ^7.0 → ^8.0
    Supports changes in Laravel 9
  • laravel/framework: ^8.83 → ^9.52.15
    Latest Laravel 9.x with security and feature updates
  • nunomaduro/collision: ^4.0 → ^5.0
    Improved error handling and Laravel 9 compatibility
2. ❌ Removed Packages:
  • facade/ignition: ^1.11
    Replaced by nunomaduro/collision in Laravel 9
  • phpro/grumphp: ^1.0
    No longer compatible with Laravel 9.x
  • roave/security-advisories: dev-master
    Laravel 9 includes built-in security tools

✅ Conclusion

This upgrade process ensures that your project is fully compatible with Laravel 9.52.15, while also updating necessary packages and removing deprecated or unnecessary ones. By following these steps, you can leverage the new features, improvements, and better performance provided by Laravel 9.52.15, ensuring enhanced security, maintainability, and a smoother development experience.
